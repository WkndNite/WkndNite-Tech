创作过程只有文字形式可不是很好玩，所以我们需要各种途径美化一下视觉效果。当然，Starlight 支持使用 MDX 和 Markdoc 文件中的组件，并提供了一些常用组件供你使用。

:::note
本页面一些效果既可以通过语法实现，也可以引入组件实现，比如你正在阅读的旁白效果。
:::

## 写作语法

---

### Markdown / MDX 语法

Markdown 是一种轻量级标记语言，它允许你使用纯文本格式来编写文档。MDX 是它的扩展语法，允许你在 Markdown 文档中使用 JSX 语法来编写组件。因此在二者文件内部都可以使用 markdown 语法引入代码块、列表、链接、引用区域、图片等。对 Markdown 语法不熟悉的同学可以参考 [Markdown 语法](https://markdown.com.cn/basic-syntax/)。而对于二者对组件的使用却各不相同，在组件部分我会详细阐述。

:::note
Starlight 集成了实验性 Markdoc 支持，但我不予以讨论。
:::

### 代码块

至于为什么我要单独讲一下代码块部分，Starlight 通过插件集成可以让你在 Markdown 默认的代码块语法里进行文本标记和窗口配置。

```js title="代码块" {1-1} {3-4} "某个词语" /正.*式/ ins="插入词语" del="删除词语"
console.log("这里是标记一行文本的演示");

console.log("这里是标记多行文本的演示");
console.log("这里是标记多行文本的演示");

console.warn("这里是标记某个词语的演示，也可以设置正则表达式");
console.log("这里是插入词语或删除词语的演示");
```

```diff lang="js"
+	console.log("这里是插入一行文本的演示");
-	console.log("这里是删除一行文本的演示");
```

```bash title="终端窗口"
echo "这里是终端窗口的演示 用于和下面进行对比"
```

```bash frame="none"
echo "这个代码块即使使用了 bash 语言也不会被显示成终端窗口"
```

### 旁白

旁白（也称为“警告”或“标注”）对于在页面的主要内容旁边显示辅助信息很有用。

Starlight 提供了一个自定义的 Markdown 语法来渲染旁白。旁白块使用一对三个冒号 ::: 来包裹你的内容，并且可以是 note，tip，caution 或 danger 类型。

你可以在旁白中嵌套任何其他 Markdown 内容类型，但旁白最适合用于简短而简洁的内容块。

## 组件使用

---

### 组件使用的配置

因为本网站不讨论对 Markdoc 的支持，因此对于想要使用组件的部分，请将文件格式设置为 `.mdx`，这样就可以通过简单的 ESM 语法使用组件了。

:::note
由于 Starlight 由 Astro 提供支持，因此你可以在 MDX 文件中添加由 受支持的 UI 框架（React、Preact、Svelte、Vue、Solid 和 Alpine） 构建的组件。
:::

```mdx

<Icon name="star" />
<Icon name="starlight" label="Starlight 的 logo" />
<Icon name="star" color="goldenrod" size="2rem" />

```

<Icon name="star" color="goldenrod" size="2rem" />
<Icon name="rocket" color="var(--sl-color-text-accent)" />

:::caution
当未设置 label 时，该图标将在无障碍下完全隐藏。 在这种情况下，请确保在没有图标的情况下，上下文仍然可以理解。 例如，仅具有图标的链接 必须 包含 label 属性才能易于识读，但如果链接已经包含了文本，并且图标纯粹是装饰性的，那么省略 label 也许更有意义。
:::
